Docker安装MyNodeQuery探针

2022-01-10

# MyNodeQuery

https://hub.docker.com/r/jaydenlee2019/mynodequery

## 简介

MyNodeQuery可以说是代替NodeQuery的个人版，一款简洁好用的探针，没有多余的元素，延续NodeQuery的生命。

演示站点：https://status.idcoffer.com

## 使用

### 依赖

- MySql 5.7 或更高版本；

- Redis （可选）

- Docker

### 安装

目前仅提供docker版本

```console

docker run -d --name=mynodequery -p 5000:5000 jaydenlee2019/mynodequery:latest

```

#### 如何解决每次更新都要从新Install的问题？

\1. 创建一个文本

​

```

/etc/mynodequery/appsettings.json

```

，内容为：

```json

{

​ "Logging": {

​ "LogLevel": {

​ "Default": "Information",

​ "Microsoft": "Warning",

​ "Microsoft.Hosting.Lifetime": "Information"

​ }

​ },

​ "Redis": {

​ "Configuration": ""

​ },

​ "MySql": {

​ "ConnectionString": ""

​ },

​ "AllowedHosts": "*",

​ "Installed": "false",

​ "ReadNodeIpHeaderKey": "X-Real-IP"

}

```

\2. 在docker run 语句中加入 -v /etc/mynodequery/appsettings.json:/app/appsettings.json 即可解决。

## 其他

### 更新注意事项

如果是从1.0.0.1或更早版本升级，请在MySql客户端执行以下SQL语句：

```sql

alter table md_nodes modify column DeleteTime datetime;

alter table md_nodes modify column LastUptime datetime;

alter table md_nodes modify column AgentUptime datetime;

alter table md_nodes add NodeStatus varchar(10) default 'Unknown';

update md_nodes set NodeStatus='Online' where AgentUptime is not null and date_add(AgentUptime, interval 70 second) >= Now();

```
